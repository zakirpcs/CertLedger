Google Chrome / Microsoft Edge / Brave
Chromium-based browsers — use the OS trust store on Windows & macOS
  1. Download ca.crt using the button above.
  2. Double-click the downloaded file → click Install Certificate.
  3. Choose Local Machine → Next.
  4. Select Place all certificates in the following store → Browse → choose Trusted Root Certification Authorities.
  5. Click Next → Finish → OK.
  6. Restart Chrome/Edge.
Or via PowerShell (admin): 
Import-Certificate -FilePath "ca.crt" `
  -CertStoreLocation Cert:\LocalMachine\Root
  1. Download ca.crt.
  2. Double-click the file — it opens Keychain Access and imports it.
  3. In Keychain Access, find the certificate under System keychain.
  4. Double-click it → expand Trust → set When using this certificate to Always Trust.
  5. Close and enter your password to confirm.
  6. Restart Chrome/Edge.
Or via terminal: 
sudo security add-trusted-cert \
  -d -r trustRoot \
  -k /Library/Keychains/System.keychain ca.crt

On Linux, Chrome uses its own certificate database. Install via the certutil tool (libnss3-tools package).

sudo apt-get install -y libnss3-tools   # Debian/Ubuntu

# Install for the current user's Chrome profile
certutil -d sql:$HOME/.pki/nssdb \
  -A -t "C,," -n "Internal CA" -i ca.crt

# Install for Chromium
certutil -d sql:$HOME/.pki/nssdb \
  -A -t "C,," -n "Internal CA" -i ca.crt

Restart Chrome/Chromium after running these commands.

Mozilla Firefox
Firefox maintains its own certificate store — OS certificates are NOT used
Method 1 — Settings UI (all platforms)
  1. Download ca.crt.
  2. Open Firefox → Settings (Alt → Tools → Settings).
  3. Search for "Certificates" → click View Certificates.
  4. Go to the Authorities tab → click Import.
  5. Select ca.crt.
  6. Check "Trust this CA to identify websites" → OK.
  7. No restart needed.
Method 2 — Policy file (enterprise/fleet)

Create policies.json in the Firefox installation directory:

{
  "policies": {
    "Certificates": {
      "Install": [
        "/etc/ssl/certs/internal-ca.crt"
      ]
    }
  }
}

Copy ca.crt to the path referenced above. Works on all platforms.

Safari (macOS / iOS)
Safari uses the system keychain — install at the OS level
macOS
  1. Download ca.crt and double-click it.
  2. Keychain Access opens — add to System keychain.
  3. Find the cert, double-click → Trust → Always Trust.
  4. Safari trusts it immediately (no restart needed).
iOS / iPadOS
  1. Email or AirDrop ca.crt to the device, or host it at a URL.
  2. Open the file — iOS prompts to install a profile. Tap Allow.
  3. Go to Settings → General → VPN & Device Management → tap the profile → Install.
  4. Then go to Settings → General → About → Certificate Trust Settings.
  5. Enable full trust for the CA under Root Certificate Trust.
Windows — System Trust Store

Installing at the system level makes all Chromium-based browsers, Edge, and Windows applications trust the CA.

GUI Method
  1. Download ca.crt.
  2. Right-click → Install Certificate.
  3. Select Local Machine → Next (requires admin).
  4. Choose Place all certificates in the following store → Browse.
  5. Select Trusted Root Certification Authorities → OK → Next → Finish.
PowerShell (admin)
Import-Certificate `
  -FilePath "C:\path\to\ca.crt" `
  -CertStoreLocation Cert:\LocalMachine\Root
certutil.exe (admin)
certutil -addstore Root ca.crt
Firefox on Windows does not use the Windows certificate store. Install the certificate separately in Firefox — see the Browsers tab.
macOS — Keychain Access

Adding to the System keychain trusts the CA for all users and all apps including Safari and Chrome.

GUI Method
  1. Download ca.crt and double-click it.
  2. Keychain Access opens — select System keychain → Add.
  3. Find the certificate in the list and double-click it.
  4. Expand Trust → set When using this certificate to Always Trust.
  5. Close the window and enter your password.
Terminal
# System-wide (all users)
sudo security add-trusted-cert \
  -d -r trustRoot \
  -k /Library/Keychains/System.keychain \
  ca.crt

# Current user only
security add-trusted-cert \
  -d -r trustRoot \
  -k ~/Library/Keychains/login.keychain-db \
  ca.crt
Firefox on macOS does not use the system keychain. Install the certificate separately in Firefox — see the Browsers tab.
Linux — System CA Bundle

Installing system-wide makes curl, wget, and most CLI tools trust the CA automatically.

Debian / Ubuntu
sudo cp ca.crt \
  /usr/local/share/ca-certificates/internal-ca.crt

sudo update-ca-certificates

Verify: ls /etc/ssl/certs/ | grep internal

RHEL / CentOS / Fedora / Amazon Linux
sudo cp ca.crt \
  /etc/pki/ca-trust/source/anchors/internal-ca.crt

sudo update-ca-trust extract

Verify: trust list | grep "Internal"

Alpine Linux (Docker base image)
apk add --no-cache ca-certificates
cp ca.crt /usr/local/share/ca-certificates/
update-ca-certificates
Arch Linux
sudo cp ca.crt \
  /etc/ca-certificates/trust-source/anchors/internal-ca.crt

sudo trust extract-compat
Chrome on Linux needs a separate certutil step — see the Browsers tab. Firefox on Linux also has its own store — see the Browsers tab.
Developer Tools & Runtimes
Docker

Inject the CA into images at build time:

FROM your-base-image

# Debian/Ubuntu base
COPY ca.crt /usr/local/share/ca-certificates/internal-ca.crt
RUN apt-get update && apt-get install -y ca-certificates \
    && update-ca-certificates

# Alpine base
COPY ca.crt /usr/local/share/ca-certificates/
RUN apk add --no-cache ca-certificates \
    && update-ca-certificates
Kubernetes

Mount the CA as a ConfigMap or Secret:

# Create ConfigMap from the CA cert
kubectl create configmap internal-ca \
  --from-file=ca.crt=ca.crt

# Reference in a pod spec
volumes:
  - name: ca-cert
    configMap:
      name: internal-ca
volumeMounts:
  - name: ca-cert
    mountPath: /etc/ssl/certs/internal-ca.crt
    subPath: ca.crt
curl & wget
# Per-request
curl --cacert /path/to/ca.crt https://your.host/

# System-wide (after installing to OS store)
curl https://your.host/           # just works

# wget
wget --ca-certificate=/path/to/ca.crt \
  https://your.host/
Python
# requests library
import requests
resp = requests.get(
    "https://your.host/",
    verify="/path/to/ca.crt"
)

# Or set env var (affects all requests calls)
export REQUESTS_CA_BUNDLE=/path/to/ca.crt

# httpx
import httpx
client = httpx.Client(verify="/path/to/ca.crt")
Node.js
# Set env var before starting Node
export NODE_EXTRA_CA_CERTS=/path/to/ca.crt
node your-app.js

# Or in code (https module)
const https = require('https');
const fs = require('fs');
const agent = new https.Agent({
  ca: fs.readFileSync('/path/to/ca.crt')
});

# fetch (Node 18+)
import { readFileSync } from 'fs';
process.env.NODE_EXTRA_CA_CERTS = '/path/to/ca.crt';
Git
# Per-repo
git config http.sslCAInfo /path/to/ca.crt

# Global (all repos, current user)
git config --global http.sslCAInfo \
  /path/to/ca.crt

# Or append to existing bundle
git config --global http.sslCAInfo \
  /etc/ssl/certs/ca-certificates.crt
Java / JVM
# Import into the JVM truststore
keytool -importcert \
  -alias "internal-ca" \
  -file ca.crt \
  -keystore $JAVA_HOME/lib/security/cacerts \
  -storepass changeit \
  -noprompt

# Or pass at runtime
java -Djavax.net.ssl.trustStore=/path/to/truststore.jks \
     -jar your-app.jar
OpenSSL / verify
# Verify a certificate against this CA
openssl verify -CAfile ca.crt server.crt

# Test a live TLS connection
openssl s_client \
  -CAfile ca.crt \
  -connect your.host:443

# Check cert details
openssl x509 -in server.crt -noout -text
